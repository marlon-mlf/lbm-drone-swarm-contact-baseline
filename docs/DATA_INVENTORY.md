# Data and source-data inventory

This release candidate contains processed source data sufficient to inspect the manuscript tables, selected generated figures, Supplementary Table S12, and the package validators. The full production simulations can be regenerated with the scripts in `scripts/`.

The production workflow generated 320000 per-step observations. In this compact release package, those observations are represented by condition-step summaries and run-level transient metrics rather than by the full long-format per-run/per-step raw table. The full long-format file can be regenerated by the analysis pipeline and may be deposited separately if repository storage permits.

All paths below are relative to the archive root `scirep_lbm_swarm_submission_package/`.

## Core configuration and source data

- `config/production_config.json`: final production protocol used by the repository scripts.
- `source_data/pipeline_config_original_from_workstation.json`: archived copy of the workstation-side pipeline configuration used during production-data generation.
- `source_data/tables/master_attempt_log.csv`: attempt-level stability and metric records.
- `source_data/tables/scirep_condition_summary_raw_unique_stability.csv`: processed condition-level endpoint and stability table.
- `source_data/tables/scirep_stability_audit_table.csv`: compact stability audit table.
- `source_data/tables/scirep_time_series_summary_by_condition_step.csv`: processed time-series summaries by condition and step.
- `source_data/tables/scirep_run_level_transient_metrics.csv`: run-level transient metrics.
- `source_data/tables/scirep_transient_metrics_summary.csv` and `.tex`: summary of transient metrics.
- `source_data/tables/count_model_coefficients.csv` and `source_data/tables/count_model_status.txt`: exploratory count-model audit files.
- `source_data/tables/scirep_quadratic_fit_table_raw_detections.tex` and `source_data/tables/scirep_quadratic_fit_table_unique_events.tex`: descriptive quadratic-fit source tables.
- `source_data/figures/`: selected main-manuscript figure source files.
- `source_data/supplementary_figures/`: selected Supplementary Fig. S1-S8 source files.

## Compact sensitivity audit

- `source_data/supplementary_tables/supplementary_table_S12_compact_sensitivity.csv`: compact sensitivity-audit summary for Supplementary Table S12.
- `source_data/supplementary_tables/supplementary_table_S12_compact_sensitivity.tex`: LaTeX source for Supplementary Table S12.
- `source_data/supplementary_tables/compact_sensitivity_attempt_logs/*_attempt_log.csv`: per-variant attempt logs for the compact sensitivity audit.

## Code and validation

- `src/lbm_swarm_contact_simulator.py`: production simulator.
- `src/lbm_swarm_contact_simulator_sensitivity.py`: compact-sensitivity simulator with pre-clamp diagnostics.
- `src/scirep_generate_tables_figures.py`: analysis/table/figure pipeline.
- `src/validate_production_outputs.py`: validator for regenerated full production outputs in `results/`.
- `src/validate_submitted_source_data.py`: validator for the compact source-data files included in this archive.
- `scripts/00_create_environment.sh` to `scripts/07_make_checksums.sh`: reproduction helper scripts. Use `scripts/06b_validate_submitted_source_data.sh` to validate this compact release without regenerating the full production `results/` tree.

## Manuscript-support and release metadata

- `manuscript_support/main_scientific_reports_release_ready.pdf` and `.tex`: manuscript support snapshot.
- `manuscript_support/Supplementary_Material_release_ready.pdf` and `.tex`: synchronized Supplementary Information with S11 and S12.
- `manuscript_support/references_bibliography_fixed.bib`: corrected bibliography source.
- `README.md`, `requirements.txt`, `environment.yml`, `CITATION.cff`, `.zenodo.json`, `LICENSE`, `DATA_LICENSE.txt`, `MANIFEST.tsv`, and `checksums_sha256.txt`: repository-level documentation, environment files, release metadata, license files, manifest, and checksum manifest.

## Validation note

The directory `results/` is intentionally used as the default output location for regenerated simulations and may be empty in this compact release. To validate the data included in the release, run:

```bash
bash scripts/06b_validate_submitted_source_data.sh
```

To validate regenerated full production outputs after rerunning the 100-run design, run:

```bash
bash scripts/06_validate_production_results.sh
```
