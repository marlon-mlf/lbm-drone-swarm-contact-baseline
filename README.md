# Scientific Reports LBM drone-swarm contact-detection package

This archive contains the code, production configuration, selected source data, and
reproducibility instructions for the manuscript:

**Baseline contact-detection scaling in fluid-coupled periodic models of self-propelled drone-swarm proxies**

The software generates and analyzes the D3Q19 BGK lattice-Boltzmann simulations used in the manuscript. It records two contact metrics: raw repeated contact detections and de-duplicated unique contact episodes. The production protocol uses nearest-node reaction forcing, spherical drone proxies, periodic domains, and 100 stable runs per condition.

## Production design

The final production dataset described in the manuscript uses:

- grids: 100x100x100 and 130x130x130
- density parameters: 1.0, 1.8, 2.93, 4.55
- agents: 30, 40, 50, 60
- stable runs per condition: 100
- time steps per run: 100
- agent radius: 5.0 lattice units
- agent mass: 25.0 lattice units
- initial velocity components: Uniform[-0.2, 0.2]
- force coupling: nearest-node reaction forcing
- density threshold: 20.0

This gives 32 conditions and 3200 stable production simulations.

The production workflow generates 320000 per-step observations (3200 stable simulations x 100 steps). This archive includes the processed run-level logs, condition-step summaries, transient metrics, and source tables used for the manuscript figures and tables. The full long-format per-run/per-step table (`master_stable_timeseries_long.csv`) is intentionally not bundled in this compact submission package, but it is regenerated by the analysis pipeline and should be deposited separately as an additional raw-data artifact if the repository storage policy permits.

## Installation

From the archive root:

```bash
bash scripts/00_create_environment.sh
source .venv/bin/activate
```

Alternatively:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Quick smoke test

```bash
source .venv/bin/activate
bash scripts/01_run_smoke_test.sh
```

Expected output: a `results/smoke_test/` folder containing `summary.csv`, `attempt_log.csv`, and time-series CSV files.

## Reproduce the 10-run design check

```bash
source .venv/bin/activate
bash scripts/03_run_full_design_10runs.sh
```

This is recommended before the full production run because it verifies that all 32 factorial conditions complete on the local machine.

## Reproduce the full production dataset

```bash
source .venv/bin/activate
bash scripts/04_run_full_production_100runs.sh
```

This can take many hours depending on CPU, memory, and Python/Numba compilation. The script uses `--skip-existing`, so it can be restarted and will not rerun completed conditions with existing `summary.csv` files.

## Validate the submitted compact source-data package

The ZIP contains processed source-data tables and figure inputs for manuscript
inspection. It does not bundle the full regenerated `results/` tree by default.
Therefore the quickest validation command checks the source data actually
included in the archive:

```bash
source .venv/bin/activate
bash scripts/06b_validate_submitted_source_data.sh
```

Expected final status:

```text
Validation status: PASS
The compact submitted source-data package is internally consistent.
```

The compatibility wrapper below first looks for a regenerated
`results/scirep_mass25_production_100runs/` tree. If that tree is absent, it
automatically falls back to the compact source-data validator, so it also passes
on the submitted ZIP:

```bash
source .venv/bin/activate
bash scripts/06_validate_production_results.sh
```

## Direct commands without wrapper scripts

Run one condition:

```bash
python src/lbm_swarm_contact_simulator.py \
  --rho 4.55 --runs 100 --agents 60 --steps 100 --grid 100,100,100 \
  --force-scheme nearest --agent-radius 5.0 --agent-mass 25.0 \
  --v0-max 0.2 --max-attempts-factor 5 \
  --output-dir results/one_condition_rho4.55_N60
```

Run the full factorial design:

```bash
python src/scirep_generate_tables_figures.py \
  --sim-script src/lbm_swarm_contact_simulator.py \
  --output-root results/scirep_mass25_production_100runs \
  --run-simulations --runs 100 --steps 100 \
  --force-scheme nearest --agent-radius 5.0 --agent-mass 25.0 \
  --v0-max 0.2 --max-attempts-factor 5 --skip-existing
```

Analyze existing outputs only:

```bash
python src/scirep_generate_tables_figures.py \
  --output-root results/scirep_mass25_production_100runs
```

## Interpretation limits

The simulator is a research code for reproducing a controlled computational baseline. It does not include sensing, communication, collision avoidance, wall effects, rotor-resolved aerodynamics, attitude dynamics, realistic deformation/damage, or traffic-management logic. The reported quantities should be interpreted as within-protocol contact-detection statistics, not operational UAV safety estimates.

## Compact sensitivity audit

A compact sensitivity-audit workflow is included for supplementary robustness diagnostics around the production protocol. It can be run with:

```bash
bash run_compact_sensitivity_audit.sh
python3 summarize_compact_sensitivity_audit.py \
  --input-root results/compact_sensitivity_audit \
  --output-csv results/compact_sensitivity_audit/supplementary_table_S12_compact_sensitivity.csv
```

The submitted source-data package includes the generated Supplementary Table S12 files and per-variant attempt logs under `source_data/supplementary_tables/`.
